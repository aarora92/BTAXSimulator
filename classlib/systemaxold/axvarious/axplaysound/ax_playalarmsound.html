Features

The basic conditions for the ax_playalarmsound class are:

  • the SlowSPS is running
  • axclientservice and axsoundserver are running


The ax_playalarmsound class is a supplement of the axclientservice and the axsoundserver.

The ax_playalarmsound class includes following features:

  • playing of the configured alarm-sound with highest priority
  • enabling and disabling playing alarm sounds
  • choice of 4 different alarm types
  • assigning alarm sounds (.wav-files) to determinated priorities
  • configureable repeating time for playing sounds

User Interface

The ax_playsound class graphical user interface includes:

  • Graphical Representation
  • Operator Interface
1. Graphical Representation

The circles next to the edit-values denote the state of the play_sound# variables:
If the circle is filled green, the sound associated to the defined and existing priority is played. Otherwise the defined priority is not existing respectively the alarm sound is not played.


Figure 1: Graphical Representation
 
 

2. Operator PopUp
 

The Operator PopUp is shared into three sections:
 

  • Display

    • There you have to enter the hostname of the client, where the alarm sounds should be played. If you make no entry the alarm sounds will be played on the server.

  • Repeat Time

    • There you have to enter the repeat time in seconds. After the configured repeat time, the alarm sound will be played again.

  • Enable Sound

    • Enables the sound-playing at the selected alarm
     
  • Prio
          This is the alarm priority to check
  • Wav
          This is the sound-file which should be played

  • repeat Sound

           If this checkbox is selected, the sound will be played again after the configured 'repeat time'.


Figure 2: Operator Popup

Implementation Notes

Class - Basic functionality

The basic funtionality of the class is, to play the high priorest alarm sound on a specified client.
The class logic starts at alarm configuration1 and ends at alarm configuration4. The alarm priorities of the configurations must be descending.

SlowSPS - Basic condition

The SlowSPS has to run as a condition for the executability of the ax_playalarmsound class.
For checking the state of the SlowSPS you have to open the Popup of the Soft PLC Panel at the aX-Server window.
If the SlowSPS is not running you have to enter the following entry in the UserProcesses.pls:
:name=slowsps autostart=True restart=True
 

Storage directory - .wav-files

The .wav-files have to be stored in the following directory:
.\automationXClientService\bin\sounds

Features

The System panel displays information collected from the server. It features the following:

  • Server Host name and status
  • Current automationX version
  • Lisence and point usage display
  • Disk Space and Memory usage
  • Adjustable Warning and Alarm indications for low space and memory
  • Adjustable hysteresis for space and memory alarms
  • Server status reports (Trending)
  • Online Help

User Interface

Graphical Representation

The GR (Graphical Representation) resides in the aX Server Window. Information such as server hostname and status, the current automationX version number, license type and point usage are available at the top of the GR, Disc Space and Memory Usage - plus any alarms for each - in the lower portion.

The software version (aX Version) indicates the automationX release number that is currently running including any service pack numbers. License information indicates the type of license currently installed, the number of points being used by the system and the number of points remaining (in the license).

Disk Space indicates physical memory usage on the hard drive where aX is installed. The LED display turns yellow when hard drive space is low and a warning is sent to the Alarm List in the AX-Viewer. When hard drive space has breached minimum level, the LED will turn red and an alarm is sent to the Alarm List. These tolerances are set up in the Server Information Pop-up.

Server Information Pop-up

This window is activated by clicking the host name on the System Class graphical representation. Hint: move the mouse pointer over the name and a button will appear.

The “Server Information” pop-up displays detailed "License Information", the amount of space used, free and total for “Working Space“ and “Trend Space”, “Total Memory” and “Real Memory”. Each resource has adjustable “Warning” and “Alarm” settings, and adjustable Hysteresis. Trending is also available via the "Trend" button and online help via the "Help" button.

License Info presents specific information about your automationX license. The State is the type of license being used, e.g. "Demo Version", "Server". The Host ID is an automatically assigned number that uniquely identifies your system.

The Points Used are the number of I/O points currently used by the system and Free indicates how many points are left open on the current license.

Working Space and Trend Space both indicate used, free and total space on the hard drive. They are separated so that different Warning and Alarm levels can be set for each. When hard drive space is above the Warning level, the status bar is green. It turns yellow and red when Warning and Alarm levels (respectively) are breached.

Total Memory is the physical RAM plus the cached disk memory added together; Real Memory is the RAM only.

Adjustable Hysteresis space is available for both Working/Trend Space and Total/Real Memory. Hysteresis is an amount of memory added to the Alarm threshold. The system will only go into alarm if the Alarm threshold itself is breached, however, in order for an alarm to turn off the system must be raised above the Alarm plus the Hysteresis amount.

 

The Trend button at the bottom-left of the popup window activates the Trend Viewer. The aX System Class is pre-configured to trend the following variables:

  • working_used: used hard drive space
  • trend_used: used hard drive space
  • phys_mem_used: used total memory (RAM + chached disk memory)
  • page_mem_used: used RAM
  • licTotalPoints: total points for the current license
  • licTotalPoints_free: total unused points for the current license

Implementation Notes

The aX System class is an integral part of the "axsystem" program. In terms of implementation it is required to have the "axsystem.tree" program installed into the running project. I.e. in the "Configuration" directory of the Project Window.

Graphic Interface

 

The PID user interface includes a process picture representation and several popup menus. The PID process representation is shown in Figure 3. The PID popup menus include the PID operations interface Figure 4, and the PID tuning interface Figure 5.

The PID process picture has a unique name identifier ($Instance) for the PID controller displayed at the top of the picture. Within the circle the setpoint value (setpoint_visual) and the process value (value_visual) are displayed. The setpoint is the top value and the text is displayed in green. The bottom value is the process value and the text is displayed in black.

Figure 3: PID Process Representation

 

To the left and slightly below the $Instance name, a symbol indicating the freeze condition of the PID controller is shown. If the PID controller is in freeze (freeze = 1) mode a "Red Minus" symbol is displayed. If the PID controller is not in freeze mode the symbol is not displayed. In addition, to the right and slightly below the $Instance name a hand symbol is displayed if the PID controller is in manual mode (auto = 0). If the controller is in auto control the hand symbol is not displayed.

Using the right mouse button to select the PID process representation causes the Operations Interface window, Figure 4, to be opened on the computer screen.

Figure 4: PID Operations Interface

The PID Operations graphic also includes the Instance name or the PID controller and below it there is a comment describing the controller.

Below the Instance name and comment, a box indicates whether the PID controller is Enabled or Locked. If the controller is enabled (enable = 1) the box background is highlighted in green and the text is shown "Enabled". The PID controller functions normally if it is enabled. If the PID controller is not enabled the box background color is red and the text shown is "Locked". In the locked mode the control output (control_out) is set to the control minimum value (control_min) which is typically 0%.

The setpoint, the process value and the control output are displayed on the three slider bars. Both the setpoint and control output can be manipulated using the slider bar up and down arrows. The setpoint and the control output can also be changed using the slider. The setpoint, process value and control output are also displayed under the slider bars.

Beneath the slider bars there are four push buttons. The Manual push button places the PID controller into Automatic mode or into Manual mode. In automatic mode the PID control output is determined by the PID logic while in manual mode the control output is set from the output slider bar.

The Manual - Setpoint push button permits the operator to adjust the setpoint from the slider. In Automatic - Setpoint mode the PID controller receives the setpoint from an outside source. The controller gets its setpoint from a variable input (setpoint_in). Placing the controller into Automatic-Setpoint allows this controller to be used in a series of cascaded controllers or as a feedforward controller.

The Shutdown push button causes the PID controller to set the setpoint to 0. The Freeze push button freezes and unfreezes the PID controller.

Finally the Parameter push button opens the PID controller parameter popup menu (Figure 5).

Figure 5: PID Tuning Interface

 

The Tuning Interface has three panes for tuning the PID parameters. The check boxes in the PID-parameters pane permits the different components of the PID controller to be enabled. The possible combinations are P (proportional), I (integral), D (derivative), PI, PD, and PID. Using the corresponding slider bar, one can adjust the controller Gain, Reset and Derivative tuning constants.

The Runup/Rundown Ramp Time modifies the maximum time that the setpoint or control output can ramp from its minimum value to its maximum value. The parameters are:

  • tu_sp = maximum time for the setpoint to ramp up from the minimum value to the maximum
  • td_sp = maximum time for the setpoint to ramp down from the maximum value to the minimum
  • tu_con = maximum time for the output to ramp from the minimum value to the maximum
  • td_con = maximum time for the output to ramp from the maximum value to the minimum
    •

    The time constants are used to set the maximum amount that the setpoint or output can change from one PLC cycle to the next. The relationship is:

    Increment = (Max_Scale – Min_Scale) * PLC_cycle_time/ramp_time              (1)

    The parameters in the Limits panel are:

     

    control_max = Maximum control output
    control_min = Minimum control output
    integral_off = If the error (setpoint minus the process value) is greater than the integral_off limit, the integral portion of the controller is disabled (anti-reset windup)
    Smoothing_time = The smoothing time constant is the time period over which the measured value is averaged (rolling average) for both PID calculations and display.
    Deadband = The deadband is the minimum error whereby no control action is taken. This prevents the control output from oscillating when the error is small.
    td_d = The delay time of the derivative

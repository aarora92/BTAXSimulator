PID Class Components

 

The components of the Regulatory PID Controller class are to be contained in the AutomationX class library, particularly in a directory named "pid_lag".

The Regulatory PID Class components consist of the Class Color file, the Class Symbol file, the HTML document files and the pixel map (graphic) files. Table 1 lists the class components and the corresponding file names.

Item Class Component Description File Name
1 Color File Defines the local color for the class pid_lag.color
2 Class Symbol File Includes the process picture representation and any popup menus pid_lag.symbol
3 HTML Document HTML document and help file for class pid_lag.htm
4 Pixel Map Files Graphic Files man.xpm
locked1.xpm
haken.xpm
Table 1: PID Class Components

The "pid_lag.color" file contains the PID class color definitions. This file holds all defined class color schemes. The PID defined colors are:

Color Default RGB Values
BorderColor 77/155/133 (dark green)
SetpointColor 0/142/0 (light green)
ValueColor 0/0/146 (dark blue)
ControlColor 142/0/0 (dark red)

 

The "pid_lag.symbol" file contains the Regulatory PID class definition for AutomationX. This file holds the user interface, the pop-up menus, the class parameters, and the PID logic.

The supporting graphic files include "man.xpm" which is the manual hand symbol, "locked1.gif" which is the freeze symbol, and "haken1.bmp" which is the check mark symbol for the PID parameters selection.

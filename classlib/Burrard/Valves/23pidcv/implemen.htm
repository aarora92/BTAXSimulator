Implementation Notes

This section briefly details how to implement the PID controller in AutomaionX. A detailed explanation of the Regulatory PID features and internal control logic is explained in the Internal Control web pages. An overview of the PID algorithm configured in AutomationX is shown in the PID Functional Diagram.

This section is divided into the following sections: Input/Output Links, Simulation Mode, and Not Connected Parameter. These sections detail the important features and the configuration of these features to properly implement the Regulatory PID controller in AutomationX.

  1. Input/Output Links

The Regulatory PID Controller consists of three inputs and one output. The three inputs are value_in, setpoit_in, and Dv. The value_in parameter is the measured value that we want to control. This value is usually a raw input signal from the field and a typical input device would be an Allen-Bradley Analog Input Flex I/O Module. The raw input signal is normally a 4 to 20 mAmp signal. The PID control block automatically normalizes the raw input signal into its engineering units.

The setpoint_in parameter is the target value that we want the value_in parameter to be controlled at. In a normal PID control loop the setpoint_in parameter is the supplied by the operator. He enters this setpoint value from the PID Operations Interface. However, this PID control loop can be configured as a Cascaded controller. The output from another Regulatory PID control loop can be used as the setpoint input to the Cascaded controller, see Figure 4.

The Dv parameter is control disturbances that is applied to the output control signal. Normally one never applies any disturbances to the control output. The default value is 0. If this value is set at a constant value of 10%, the minimum control output would be set at 10%. This Dv parameter could be used to set a minimum bias on the control output.

The control output parameter is control_out.

  1. Simulation Mode

A project in AutomationX can be placed into simulation mode. If the global simulation parameter ($$_simulation) is set ON, the project is placed into simulation mode. In simulation mode all the input and output connections to I/O is disabled and alias entities are substituted in its place.

Using the simulation feature in AutomationX, one can simulate control and logic strategies before implementing it on a real system.

  1. Not Connected Parameter

During the PID loop configuration the I/O for the controller may not have been set up. To avoid PLC runtime errors when the project is installed, the Not Connected (NC) parameter for the PID filed inputs and outputs should be enabled. The NC parameter is located in the editor window for the PID parameter configuration. Select the input or output parameter and the NC enable is located at the bottom of the editor window.

When the I/O is installed in the project do not forget to disable the NC parameter. If this is not done the I/O will not receive any control.

Profibus Status Popup

With a mouse click on an active Profibus card slot on the MIO_PCI panel you will get the Profibus Status Popup.
 


In the Profibus status popup you can see the protocol name of your Profibus connection in the field PB-SW, the status string (e.g. In Ring) in the field PB-Status and the running counter of you Profibus connection in the field PB-RunCnt. You can see also the number of active slaves in the field Active Slaves, the sum of input data in the field Input Data and the sum of output data in the field Output Data.
With the button Help you will get this online documentation of the Profibus setting, with the button Parameter you will get the Profibus Parameter Popup and with the button Exit you can close the popup.
 
Status field color Description
Green Communication okay
Red (flashing) Communication not okay
Light Grey No Card

If your Profibus card doesn't run, your Profibus running counter will stop, and the staus field color will flash red.
 

Profibus Parameter Popup

If you press the Parameter Button you will get the Profibus Parameter Popup.
 


In this popup you can change all the parameters of you Profibus communication.

address = master address

slot time = the maximum time a master station on this Profibus must wait for a transaction response.
The Slot time is the maximum time the initiator waits for the complete receipt of the first frame character of the immediate acknowledgment or response, after transmitting the last bit of an action frame.
Default value: 200 TBit

tsdr_min = the minimum time a slave must wait before it may respond.
Default value: 60 TBit (in case of the microprocessor 8051 --> normal value 11 TBit )

tsdr_max = the maximum time after wich a slave must respond.
Default value: 500 TBit (in case of the microprocessor 8051 --> normal value 100 TBit )

ttr =Target Rotation Time
the anticipated time for one token round on this Prufibus, including allowances for high and low priority transactions, errors and GAP maintenance.
Default value: 60000 TBit

baud rate
The available baud rates of the MIO PCI Profibus card are 1500 kBit/sec, 500 kBit/sec, 187,5 kBit/sec, 93.75 kBit/sec, 19.2 kBit/sec  and 9.6 kBit/sec.

gap = range of station addresses from this station to its successor in the local token ring, excluding stations above hsa.
GAP-factor: Specifies the number of bus cycles after which the master looks for new active stations to accept them in the bus and pass on the token to  this station. The GAP area is between the own station address and the next one. Execption: The arena starting with the highest station address and ending with address 126 is not part of the GAP arena.
Default value: 1

tset = setup time
The time between an event and the necessary reaction.
Default value: 1

hsa = highest station adress of masters in bus system
Default value: 2
Max value: 127
Min value: 1
 

Load and Save Settings:

With the buttons Load Prm and Save Prm you can load or save your Profibus settings from a file. The path and the filename of this file you can choose by a file selection box which will  be invoked by a mouse click on the field file name.
Your automationX system also includes a file with the default Profibus settings.

Reset Profibus Communication:

If you have changed a Profibus parameter in the popup, the button Reset --> Set new PB Parameter will start to flash red. Only when you press this button your Profibus communication will be reset and the new parameters will get active.

Bus Terminator:

The bus cable shall be terminated at both ends. The termination resistor Rt shall be completed by a pull-down resistor Rd (connected to Data Ground DGND) and by a pull-up resistor Ru (connected to Voltage-Plus VP). This supplement forces the differential mode voltage(i.e. the voltage between the conductors) to a well defined value when no station is transmitting (during the idle periods).

The MIO PCI Profibus card is producted with internal bus termination. You can switch the internal bus termination on or off with a dip-switch on the card.

Implementation Notes

The axs_mio_pci class is an integral part of the "axsystem" program. It is required to have the "axsystem.tree" program installed into the running project, i.e. in the "Configuration" directory of the Project Window.

The instance name of the automationX Profibus PCI card class must correspond to the WinNT driver registry keys. To start up the registry editor select "Start" - "Run" from the WinNT task bar and type "regedit" or "regedt32".

These keys can be found under the following path: HKEY_LOCAL_MACHINE/SOFTWARE/automationX/Initialize (i.e. RegKey:PCI_1 - Instance name: PCI_1).

This path only exists if the automationX PCI Card Driver is installed correctly (Installation or reinstallation required).

The card type is displayed in the Registry Variables "MIO_0" and "MIO_1" which might have the values displayed in the following part.

automationX Profibus PCI Card: < /EM>

MIO_<X> = 10:  The corresponding card module <X> is a Profibus Redunancy channel.

MIO_<X> = 13:  The corresponding card module <X> is a Profibus DP channel. < /p>

automationX Serial PCI Card:

MIO_0 = 3: The card module 0 is a four channel Serial Communication Module.

MIO_1 = 0: No second module available.

i.e.: WinNT Registry Key shows :

  • PCI_1: MIO_0=10, MIO_1=13 - This is an automationX PB PCI Card (Module 0 is used as a Profibus Redundancy channel, Module 1 is used as a Profibus DP Channel).
  • PCI_2: MIO_0=3, MIO_0=0 - This is an automationX 4 channel Serial PCI Card.

The used pc slot is displayed in the Registry Variable "PcSlot". Its value depends on the used motherboard. 

Note: If the PCI Card(s) get(s) replaced on the motherboard or the "MIO_<X>" values do not match with the values shown in the part above, the driver must be reinstalled.

After assingning the instance name, select the specified card from the enumerator "card", which can be found under the server data tab in the object explorer.

If the instance name is not the same as the name required by the system, or if the "card" resource selection does not match with the instance name, a red flashing bar with "Wrong Instancename" is displayed.

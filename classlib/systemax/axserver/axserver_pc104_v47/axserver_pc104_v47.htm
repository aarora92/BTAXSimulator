PC104_Tools.axx  

Features

The PC104_Tools.axx library is a collection of many functions to control the PC104 aXcontroller board. The library includes functions for follow devices:

  • the display
  • the display-keys
  • the display-background lighting

The Display

The display has 8 virtual screen pages, in witch you can draw. When you will put the grafic on the display you must "Update" a virtual page. The display has 128*64 pixels. The top-left corner has the coordinates (0,0) and the bottom-right corner has the coordinates (127,63).  

( PC104_Tools overview )


There are several draw functions: 

SetPixel draws a black pixel.
ClrPixel draws a white pixel.
XorPixel inverts a pixel.
Line draws a line.
Box draws a rectangle or a frame.
Put draws a bitmap image. 
PrintText draws a text. 


And the four special functions:

SetPage selects a virtual page.
Update copy a virtual page to the display.
SetMode sets the drawing mode for "Put" and "PrintText".
SetFont sets the font for "PrintText".

The "SetPage" Function

With the "SetPage" function you can select the virtual Page in witch you can draw respective the page witch should put on the display with the "Update"  function. There are 8 virtual pages to select, from 0 to 7.

Use:

  axx_function("PC104_Tools","SetPage",page);          

page is the selected virtual page (0-7).   

The "SetPixel" Function

With the "SetPixel" function you can set a black pixel on a position.

Use:

  axx_function("PC104_Tools","SetPage",x_pos,y_pos);          

x_pos is the selected x-coordinate (0-127).   
y_pos is the selected y-coordinate (0-63). 

 

The "ClrPixel" Function

With the "ClrPixel" function you can set a white pixel on a position.

Use:

  axx_function("PC104_Tools","SetPage",x_pos,y_pos);          

x_pos is the selected x-coordinate (0-127).   
y_pos is the selected y-coordinate (0-63). 

 

The "XorPixel" Function

With the "XorPixel" function you can invert pixel on a position. If a black pixel is on the selected coordinate, it turns to white. If a white pixel is on the selected coordinate, it turns to black.

Use:

  axx_function("PC104_Tools","XorPage",x_pos,y_pos);          

x_pos is the selected x-coordinate (0-127).   
y_pos is the selected y-coordinate (0-63). 

The "Line" Function

With the "Line" function you can draw a line in a virtual page. 

Use:

  axx_function("PC104_Tools","Line",x_start,y_start,x_end,y_end,color);       

x_start is the selected x-coordinate form the begin of the line (0-127).   
y_start is the selected y-coordinate form the begin of the line (0-63). 
x_end is the selected x-coordinate form the end of the line (0-127).   
y_end is the selected y-coordinate form the end of the line (0-63). 
color is the draw color ( 0=black 1=white 2=invert ).

 

 

The "Box" Function

With the "Box" function you can draw a rectangle in a virtual page. You can draw it full filled or only the frame.

Use:

  axx_function("PC104_Tools","Box",x_start,y_start,x_end,y_end,color);          

x_start is the selected x-coordinate form the begin of the line (0-127).   
y_start is the selected y-coordinate form the begin of the line (0-63). 
x_end is the selected x-coordinate form the end of the line (0-127).   
y_end is the selected y-coordinate form the end of the line (0-63). 
color is the draw color ( 0=black 1=white 2=invert 128=black-frame 129=white-frame 130=invert-frame).

 

The "Update" Function

With the "Update" function the data of the selected virtual page will copied to the display. 
Attention ! Don't use this function in the SPS_Cycle . The function needs between 1 ms to 100 ms execution time.

Use:

  axx_function("PC104_Tools","Update");          

The "Put" Function

With the "Put" function you can put a bitmap image on a seleced position. Before you can put an image to the virtual page. you must registrer it in the "./automationX/PC104/PC104_Tools.ini" file. With the "SetMode" function you can select the colors for the image. ( one to one, inverse , transparent ... )

To register a image make follow steps:

  1. Draw a 2-color bitmap wiht your grafic program (e.g. CorelPhotoPaint or Paint)
  2. Save the bitmap as a "*.bmp" file.
  3. Start the "./automationX/bin/BitmapConvert.exe" program.
  4. Take as source path the name of your bitmap, and insert as target path (e.g.  "./automationX/PC104/Bitmaps/TestImage.bbm" )
  5. Press "Convert", now a new "*.bbm" file must be inserted in the "./automationX/PC104/Bitmaps" folder.
  6. Open the "./automationX/PC104/PC104_Tools.ini" file with Notepad.exe  editor.
  7. Write an new line with the name of your bitmap in the file. ( e.g. Bitmap 1 "TestImage.bbm" ) The number behind the Bitmap word is the image number. You can use the number 1 to 63 for your images. You can use only one number for each image.
  8. Save the "./automationX/PC104/PC104_Tools.ini" file.
  9. Start automationX new.
  10. Now the image is ready for use. 

Use:

  axx_function("PC104_Tools","Put",x_pos,y_pos,image_nr);          

x_pos is the selected x-coordinate (0-127).   
y_pos is the selected y-coordinate (0-63). 
image_nr is the number of the image witch should put in the page. (0-63). The number 0 is for the automationX-logo and the number 1-63 ist used for user images. (registered images). 

The "PrintText" Function

With the "PrintText" function you can print a text on a seleced position. With the "SetFont" function you can select the font and the font-size. With the "SetMode" function you can select the colors for the font. ( one to one, inverse , transparent ... )

Use:

  axx_function("PC104_Tools","PrintText",x_pos,y_pos,text);          

x_pos is the selected x-coordinate (0-127).   
y_pos is the selected y-coordinate (0-63). 
text is the string wich should be displayed. You can insert a variable form automationX ( Varname ) or a string ( "Your Text" ) .

The "SetFont" Function

With the "SetFont" function you can select the used font and the font size.

Use:

  axx_function("PC104_Tools","SetFont",font_nr,font_size);          

font_nr is the selected font:

0 = Arial
1 = Courier
2 = Georgia
3 = TimeNewRoman
4 = Verdana
5 = User defined Fonts

font_size is the size of the font. Only the sizes 8,10,12,14 are aviable. For "User defined Fonts" all sizes are aviable.

 

To register a user defined font make follow steps:

  1. Start the "./automationX/bin/FontCreate.exe" program.
  2. Insert as target path (e.g.  "./automationX/PC104/Fonts/TestFont.fon" )
  3. Press "Create Font", and select your font. 
  4. Press "OK" and a new "*.fon" file must be inserted in the "./automationX/PC104/Fonts" folder.
  5. Open the "./automationX/PC104/PC104_Tools.ini" file with Notepad.exe  editor.
  6. Write an new line with the name of your font in the file. ( e.g. Font 1 "TestFont.fon" ) The number behind the Font word is the font size number. You can use the number 1 to 15 for your fonts. You can use only one number for each font.
  7. Save the "./automationX/PC104/PC104_Tools.ini" file.
  8. Start automationX new.
  9. Now the font is ready for use.

Use Keys 

The aXController has four buttons on his front panel. With the function "Keyboard" you can read the state of the keys. The return value of this function is a 32-bit value. The bits of this value have follow definitions:

Bit

Use

0

high if key 1 is pressed (the left button)

1

high if key 2 is pressed (the middle left button)

2

high if key 3 is pressed (the middle right button)

3

high if key 4 is pressed (the right button)

8

high if key 1 was pushed

9

high if key 2 was pushed

10

high if key 3 was pushed

11

high if key 4 was pushed

16

high if key 1 was let off

17

high if key 2 was let off

18

high if key 3 was let off

19

high if key 4 was let off

Bits 0-3 are high when the key is currendly  pressed.
Bits 4-11 are high when the key was pressed the first time.
Bits 16-19 are high when the key was let of.  

Use:

 key=axx_function("PC104_Tools","Keyboard");

Use background lighting 

 With the function "Light" you can turn on or off the display backgrount lighting.

 

Use:

 key=axx_function("PC104_Tools","Ligth",on_off);        

on_off is the state for the ligth (0=off 1=on).

Use XEA_Text 

 With the function "XEA_Text" you can write 2 lines of text in the XEA500 Display. 

Use:

axx_function("PC104_Tools","XEA_Text",0,0,XEA_TEXT1);

axx_function("PC104_Tools","XEA_Text",1,0,XEA_TEXT2);

XEA_Text(ROW,COL,TEXT) ROW=0..1 COL=0..15 TEXT=ASCII-String

Use XEA_Keyboard()

 With the function "XEA_Keyboard()" you can recognize wheter theres a KEY pressed. 

Use:

XEA_KEY = axx_function("PC104_Tools","XEA_Keyboard");

XEA_KEY is bit-encoded.

Keys :

UP: 16;

DOWN: 4;

LEFT: 2;

RIGHT: 1;

ENTER: 8;

Use XEA_Leds()

 With the function "XEA_Leds()" you can set the 4 LEDs on the XEA500. 

Use:

axx_function("PC104_Tools","XEA_Leds",XEA_LED);

 

LEDs:

XEA_LED.0: RUN

XEA_LED.1: READY

XEA_LED.2: TEMP

XEA_LED.3: MASTER

 

If 1 the LED will be RED

IF 0 the LED will be GREEN

There is NO LED OFF (Black) possible.#

Use XEA_Address()

 With the function "XEA_Address()" you get the station address of the XE500. 

Use:

XEA_HEXADDRESS = axx_function("PC104_Tools","XEA_Address");

 

In the lower Byte you receive the Info of the station-address.

In the higher Byte you receive the Positioninfo (Left, right, Standalone)

Front Display

The display indicates the actual state of the very XEA-Unit.

The first line shows the hostname of the XEA.

The second line shows the actual state of the arbitration.

That can be :

Standalone if the XEA is working Standalone

Main Server if the XEA is working as Master

Stand-By Server if the XEA is working as Standby

Stand-By (wait) Server if the XEA is working as Standby and is doing a redundancy synchronization

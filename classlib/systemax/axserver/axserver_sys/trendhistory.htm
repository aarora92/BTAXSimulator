Trend History

Trend History is a function to save old trend files into an adjustable directory. The directory could either be a folder on your local system or a network- drive. You can open the Trend History configuration pop-up by using the button Trendhistory (Server Information).

Basic Settings

Checking the hostnames

After the implementation of Trend History the hostnames will be assigned to the paths. This will happen full automatically. Furthermore the occurrence of the pop-up shows you, if you sit in front of a stand alone server, the main- or the stand-by server.

 

On the example above you can see that only one hostname is visible. Therefore you know, that you are using a stand- alone server.

 

In this case two hostnames are visible. Therefore you know, that you are using two redundant servers. If you see a green led in front of the hostname, you know, that you are currently working on the main- server. A yellow led indicates, that you are working on the stand-by server. If both led's in front of the hostnames are grey, you have to refresh the pop- up by pressing the button Read Servernames, otherwise Trend History would not work. If the button seams not to work, make sure that you have started AutomationX correctly. It is necessary that you enter the hostnames case sensitive (for example: bin\flash -server DELLTEST3).

Path syntax

Example:

You want to save your data to: C:\trendhistory\test1

Therefore you have to use following syntax: \C=\trendhistory\test1

Using a network drive

If you want to use a network-drive, you have to create a connection manually. For example, if you use windows, you have to open the workspace and create a connection to the requested drive. Please make sure, that you have full permissions to this drive (you need to write into the directory).

Setup possibilities

You can use Trend History in different ways:

·  Disable Trend History

·  One stand-alone server saves on his local drive

·  One stand-alone server saves on a network-drive

·  Two redundant servers, both save on their local drive

·  Two redundant servers, both save on the same network drive

 

Disable Trend History

There are two ways to disable the Trend History function:

1.)  You disable all 3 flags in the pop-up

2.)  You delete both paths

After you disabled Trend History make sure, that the leds in front of Trend History active are grey on both servers. This ensures that the Trend History function is disabled.

One stand-alone server saves on his local drive

Enter a local path into the control array of path1 and set the relating flag. If there are two green leds, one in front of Trend History active and the other one in front of the hostname, Trend History is working.

One stand-alone server saves on a network-drive

First of all, create a connection to a network- drive. Afterwards enter the path into the control array of path1 and set the relating flag. If there are two green leds, one in front of Trend History active and the other one in front of the hostname, Trend History is working.

Two redundant servers, both save on their local drive

 

Enter two local paths into the control arrays and set the flags for path1 and path2. If you open the pop-up at the main- server, you shall see two green leds, one in front of the hostname and the other one in front of Trend History active. If you open the pop-up at the stand-by server, you shall see a yellow led in front of the hostname and a green led in front of Trend History active. Now both of the servers are saving the trend data on their local drives.

Two redundant servers, both save on the same network drive

 

First of all create a connection to a network- drive on both servers. The names of these network drives have to be the same at both servers. Afterwards you have to enter the path of the network drive into the control array of path1. At least you have to set the flag in front of Main Server saves Trend History Files to Path1. As you can see at the example above only the main-server is active. The stand-by server is disabled.

 

If you do a redundancy- change, the pop-ups have to look like above (the other server is active).

Features

The Soft PLC panel displays the current status of the Soft PLCand Slow PLC. It features the following:

  • Running status
  • Cycle time
  • Life tick
  • Cycle execution time
  • Peak execution detection and Reset Peak button
  • Shared memory used by automationX

User Interface

Soft PLC Panel

The GR (graphical representation) of the Soft PLC class displays the current Soft PLC status, cycle time, last cycle execution time and the amount of shared memory (RAM) currently used by automationX processes.
 

The Soft PLC status indication displays two states:

  • Green: Soft PLC status is OK. Message reads "Running".
  • Red: Soft PLC failure. Message reads "Not Running".
The last execution time status bar displays the same two states; i.e. Red = Soft PLC Failure, and Green = Soft PLC Running.

The Shared Memory status bar displays similarly for the shared memory amount. If memory resources (RAM) on the host computer go beyond the minimum required amount for automationX processes to run, the status bar will turn Red.

Integrated PLC Pop-up

The “Integrated PLC” popup window shows specific information about the Soft PLC and Soft PLC - Slow status.

This window is activated by clicking the status indication on the Soft PLC class graphical representation. Hint: move the mouse pointer over the name and a button will appear.

The Soft PLC portion shows information about the regular Soft PLC cycle.

  • Cycle Time is the current setting (in milliseconds) between execution cycles. This setting is adjustable using the increment/decrement arrows or by selecting the value and typing in a new number.
  • Peak Detection is a read-only value. This is a record of the longest execution interval for the current running session since the last reset. This value is reset during a system re-start or by selecting the Reset Peak button.
  • Last Time is the execution time (in milliseconds) of the last cycle.
  • ID is the number of warmstarts since the system was last started.
  • Lifetick is a counter that increases by one every Soft PLC cycle from the beginning of the current session.
  • The status at the bottom of this section shows the current state of the Soft PLC. It mimics the Soft PLC status display on the class GR (graphical representation).
The Soft PLC - Slow shows informaion about the Slow PLC cycle. The Slow cycle is a secondary execution time that is used for applications that do not have to execute every PLC cycle (for details on the Soft PLC - Slow cycle, please see the automationX System Manual).
  • Cycle Time is the current setting (in milliseconds) between execution cycles. This setting is adjustable using the increment/decrement arrows or by selecting the value and typing in a new number.
  • Lifetick is a counter that increases by one every Soft PLC cycle from the beginning of the current session.
  • The status at the bottom of this section shows the current state of the Soft PLC - Slow process. If the process is OK, the display is Green with the message "Slow PLC Running". If the process has a failure, the display is Red - the message reading "Slow PLC Not Running".
Shared Memory shows resource information about the server.
  • Free is the amount of free RAM on the server computer.
  • Locked is the amount of RAM currently in use by automationX processes.

Trending is available - via the "Trend" button - for the following variables:

  • cycle_time: normal Soft PLC cycle time
  • cycle_time_slow: Soft PLC - Slow cycle time
  • time_plc_last: execution time of the last PLC cycle
  • peak_detector: longest recorded execution interval of the normal Soft PLC cycle time
  • free_space: amount of RAM currently available on the server
  • locked_space: amount of RAM being used by automationX processes

Implementation Notes

The Soft-PLC class is an integral part of the "axsystem" program. In terms of implementation it is required to have the "axsystem.tree" program installed into the running project. I.e. in the "Configuration" directory of the Project Window.

User Interface

1.Graphical Representation

Figure 1: Graphical Representation
 

2. "recordselection" Popup

With the "recordselection" Popup you can load, delete and edit the data record.
The record can be selected by the up and down buttons beside the "Data record selection" field or directly by a click on the entry with the left mouse button.
The "Load" button loads the data from the data record in the variables Data1_1 - Data20_1. The current loaded data record is visible in the "Loaded Data record" field.
With the "View Data record" button the data can be viewed without loading. The "Filename" field shows you the filename of the data record you want to see.
If you have administrator rights you can delete a data record with the "Delete Data record" button.
Further you can make a new data record. By pushing the "New Data record" button the "dataedit" Popup stands up.
 

Figure 2: recordselection Popup
 

3. "dataedit" Popup

A new data record can be created in this Popup. It includes the name and allows twenty data entries. The name must not be longer than 15 signs. With the "Save Data" button you store the new data record and it results an entry in the "recordselection" Popup (Figure 3). If you store data under a existing record name an AXMessage appears on the screen (Figure5). If you say "Yes" the existing file will be overwritten. The files are saved in the directory /Automationx/projects/cache/daten (Figure 6) under the record name plus the extension *.xls (i. e. record2.xls). The ./cache/daten directories will be created at the first time you press the "Save Data" button. A maximum of 27 data records can be saved.

Programmer Information

Description of commands for data handling:

Storage Data:   AXStoreParameter("$Instance",filenamedir,0,15);

$Instance stands instead of the Instance name. For the filenamedir stands the absolute path of the filename of the data record which should be saved. The default path is ./projects/cache/daten/filename. "filename" is the name of the data record with the extension .xls (i.e. record0.xls). 0 must be chosen if the data record should be overwritten, 1 must be chosen if the data record should be added to the last one. The 15 indicates the maximum number of data records. If more then this number are stored, the data record on the top will be deleted (like shift register).If the data are stored correctly, a return code with 1 is provided. In the case of a not successful storage, a 0 is given back:

 x=AXStoreParameter("$Instance",filenamedir,0,15);
 
 

Delete Data:   AXDeleteParameter("$Instance",filenamedir,4,0);

$Instance stands instead of the Instance name. For the filenamedir stands the absolute path of the saved filename. The default path is ./projects/cache/daten/filename. "filename" is the name of the data record with the extension .xls (i.e. record0.xls). The 4,0 in this example means, that the fourth data record, counted from the top of the list (this means the 0), should be deleted. If one wants to count from below of the list, 1 must be chosen instead of 0 (i.e. the last saved record must be chosen with1,1). If the data are deleted correctly, a return code with 1 is provided. In the case of a not successful storage, a 0 is given back:

x=AXDeleteParameter("$Path",filenamedir,0,15)
 
 

Read Data:   AXReadParameter("$Instance",filenamedir,4,0);

$Instance stands instead of the Instance name. For the filenamedir stands the absolute path of the saved filename. The default path is ./projects/cache/daten/filename. "filename" is the name of the data record with the extension .xls (e.g. record0.xls). In this example the fourth data record, counted from the top of the list, should be read. If one wants to count from below of the list, 1 must be chosen instead of 0 (i.e. the last saved record must be chosen with 1,1).

Further a data record can be searched and read in via a variable. Instead of e.g. 4,0 the column name and the searched variable must be indicated.
IMPORTANT: The whole argument must be declared under Quotation marks (e.g. "Data2=56").

TCP_RdWr

Features

The TCP Read Write symbol communicates with another TCP Read Write symbol via TCP.

One variable or whole symbols can be transmitted.

Graphical User Interface
 
The TCP_RdWr class represents an actual Read-Write module. The instance name of the

module will appear in white text on the top right of the symbol. On the top left the symbol

mode (Read / Write) is displayed. The title will change from green to flashing red if an error

occurs.

The symbol is divided into four sections:

  1. Data visualization with variable connection info:

    2. Connection: Actual connection with variable or symbol.

    Datatype: Type of connected data.

  2. Communication info with com. LEDs

    3. Write mode -> Transmission

    done: Transm, done

    OK: Transm. OK

    Trigger: Transm. trigger

    started: Transm. started

    Read mode -> Receive

    NewRec: New data received

    Overflow: Data overflow

  3. Connection info with status LEDs and Lifetick

    4. OK: TCP connection OK

    Error: TCP connection error

    Lifetick: In lifetick mode number of received lifeticks

  4. Status string

Actual status of symbol.

 

Figure 1: Graphical Representation (Write mode)

 
 

Figure 2: Graphical Representation (Read mode)

The PopUp is invoked by selecting the graphical representation. It shows the

communication settings, symbol select button, trigger button, communication LEDs,

connection LEDs and the status string. To activate the symbol the Select button must

be on. On the top of the PopUp the TCP communication settings are displayed.

The Transmission, Connection and status string sections are similar to them in the

Graphical Representation.

Implementation Notes

Symbol for data transfer of one variable or one symbol via TCP.

Supported datatypes:

BOOL, BYTE, SINT, USINT, INT, UINT, DINT, UDINT, REAL, LREAL, STRING, ALARM and ENUM

Modbus Client

Features

The Modbus Client symbol must be connected with a Modbus Master.

The Client can store up to 128 words of Master data.

Graphical User Interface

The ModbusCL class represents an actual Client module. The instance name of the module will appear  in white text on the top right of the symbol. The title will change from green to flashing red if an error occurs.

The symbol is divided into four sections:
 

- The Modbus master
        The inctance name of modbus master is displayed.

- Client information
        The inctance name of the client, the Nr. (it's the same as the array index from the client_name variable in the MB master);
        actual MB function and MB address are visualisized.

- Trigger

- State string
        Informational string.
 

Figure 2: Graphical Representation
 
 

The PopUp is invoked by selecting the graphical representation.

Polling mode can be set to trigger or cyclic. If trigger, you can send a query to a MB slave

by pressing the trigger button.

Address:

The MB slave address .

Slave Offset:

Offset within the MB slave.

Offset:

Offset in the MB Client data array.

Length:

Read / write length.

Function:

Query function to the MB slave.

Configuration

The following table lists the configuration parameters for the ModbusCL symbol. The parameter data type, the default values and a brief description are listed. The grey shaded rows are parameters which must not be changed.
 
Item Variable Type  Default  Description
1 modbus_master STRING The instance name of the Modbus master object
2 nr USINT 0 Index number from MB master
3 address USINT 0 The destination highway address of the modbus slave
4 function MB_Function Read_Coil The modbus function to execute
5 trigger BOOL NO Trigger the client block function to execute
6 poll_mode MB_mode trigger The communication method (cyclic or triggered)
7 slave_offset UINT 0 The offset within the Modbus slave
8 offset UINT 0 Offset in Words in the PLC data array
9 length UINT 0 Data length to read/write
10 data UINT 0 The array of data to store/send
11 info_string STRING 0 Information string

 

The Modbus functions:

Read_Coil:

Slave Offset in coils (bits) only whole bytes = 8,16,24,32,.. bits

Length in coils (bits)

Offset in words within the client data array.

Read_Input:

Slave Offset in inputs (bits) only whole bytes = 8,16,24,32,.. bits

Length in inputs (bits)

Offset in words within the client data array.

Read_Hold_Reg:

Slave Offset in registers (words)

Length in registers (words)

Offset in words within the client data array.

Read_Input_Reg:

Slave Offset in registers (words)

Length in registers (words)

Offset in words within the client data array.

Force_One_Coil:

Slave Offset in coils (bits)

Length not used.

Offset in words within the client data array.

Preset_One_Reg:

Slave offset in registers (words)

Length not used

Offset in words within the client data array.

Read_Exep_Stat:

Slave offset and Length not used.

Offset in words within the client data array.

Force_Mult_Coils:

Slave Offset in coils (bits) only whole bytes = 8,16,24,32,.. bits

Length in coils (bits)

Offset in words within the client data array.

Preset_Mult_Regs:

Slave Offset in registers (words)

Length in registers (words)

Offset in words within the client data array.

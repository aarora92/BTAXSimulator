MOBY ASM470

 

Generally:

For usage MOBY ASM470 a ET200 module (for example ET200M IM153-2) is necessary. The ET200 module gets a profbus address and handles the communication with the ASM470. A command consist of 4 byte header and 12 byte data, total 16 byte per command.

Two SLG's (means Schreib / Lesegerät) can be connected to one ASM. The communication with two SLG's takes place in time multiplexer mode. For more information see also the SIEMENS MOBY ASM470 documentation.

 

Graphical Representation:

The LED's show the communication state (SF, 24V) and the states for SLG1 and 2 (ANW, active, Error, SLG1, SLG2, RxD).

Status Popup:

A mouse click on the graphical representation opens a status popup.

Message

Description

"ASM Failure"

Communication to ET200 and / or MOBY ASM470 fails.

"Order active"

ASM470 executes at this time a command.

"No Order"

No command to any SLG.

"MDS in Scope": The presence control is active and a MDS exists in the read /write field of a SLG.

"NO MDS": The presence control is not active or no MDS exists in the read /write field of a SLG.

 

Parameter Popup:

Parameter

Description

ECC Mode

Error Correction Code. For more data security. For using this feature a new initialisation (after activation and reset) for the a MDS is necessary.

SLG

SLG selection.

MDS-Type

MDS type.

ANW

Means the presence control. It would be active if "SLG_with_Anw" is shown.

Moby Type

Moby type.

Timebase

Parameter for the sampling intervall. Recommended for Moby Type "MOBY-I/MDS507" or "MOBY-V".

Timevalue

Parameter for the sampling intervall. Recommended for Moby Type "MOBY-I/MDS507" or "MOBY-V".

The sampling intervall is the product between Timebase and Timevalue.

This parameter will take effect after pushing the "Reset" button.

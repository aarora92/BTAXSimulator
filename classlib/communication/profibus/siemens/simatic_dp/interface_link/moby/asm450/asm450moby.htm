Siemens ASM450

Features

This class can be used to communicate with the SIEMENS MOBY-system.


Features | Installation | User Interface | Configuration | Implementation Notes

 
 


 
 

Integrated Control: All control is fully integrated; no external PLC is required.

Field Link:The Siemens ASM450 requieres DC24V and Profibus Connection

Depression of Consecutive Alarms in case of Invalid I/O: The ASM450 Class always checks the validity of the connected I/O. In case the field link is interrupted the ASM450 depresses alarms, which would arise due to I/O status in normal operation.

Pop-Up Menus: The ASM450 Class comes with a punch of pop-up menus, which are inherited through the class; no configuration is necessary.

2. Operator Pop-Up Menu

The Operator Pop-Up Menu is invoked through a mouse click anywhere on the motor's graphical representation.

The Operator Menu comprises of the following sections:

  • Identification: the texts for tag name and remark are automatically taken from the instance's Name and Remark Field. A mouse click in this section closes the pop-up menu!
  • Status Line
  • Menu Pane for Extended Functions: such as "Class Documentation", "Instance Documentation", "Parameterization", and "Service Functions".
  • Status Indication: "Feedback" indications, "Interlock" indication, "Failure" indication.
  • Control Section: Automatic/Manual Mode, Motor On/Off in Manual Mode, Failure Acknowledge.

3. Parameterization and Mode Pop-Up

This pop-up menu is password protected!
 

  • The Parameter section allows an operator-friendly setup of the motor's maximum response time. The default value is five second.:
  • The Modes section allows switching to (local) Simulation Mode. In this mode all inputs are simulated and the output is disconnected from the field! The Overwrite Interlock Mode connects the auto_on variable with the field no matter of the status of the interlock variable. This is useful when one MUST turn on the motor despite of interlock conditions. If none of the two Check Boxes is activated the motor is in standard mode.
  • The Emulations section allows an individual emulation of the field inputs in case they are INVALID. Then the Run Input "follows" the output with a variable delay time (default = 3 seconds). The Ready and Remote feedbacks are considered to be ON.


4. Maintenance Pop-Up
 

  • The "Enable Maintenance" Button enables or disables all maintenance functions.
  • If one of the limits is exceeded the motor class launches a Service Request "Warning".
  • Each Limit can be reset individually; then the date is entered in the according field.

Configuration

1. I/O Configuration

The Motor Class has only a few configuration parameters. At first the motor must be linked to the field. This is simply done by filling out the reference fields of the following variables: on, run_in, rdy_in, rem_in. The other possible parameters are: the maximum time delay between a output command and the arrival of the proper feedback (error_time) and the limits for service request (compare maintenance pop-up).

Notes:

  • If "run_in" is not connected, the class routes the output to the internal run variable. This fakes that the feedback is always OK. If run_in is connected, the field must provide the proper feedback!
  • If "rdy_in" is not connected, the class assumes that the motor is always ready.
  • If "rem_in" is not connected, the class assumes that the remote switch is always turned on.
  • The maximum delay time for the run feedback also can be changed in online mode by means of the parameterization pop-up menu.

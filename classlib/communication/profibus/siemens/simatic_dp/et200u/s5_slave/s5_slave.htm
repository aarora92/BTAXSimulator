Features

User Interface

The s5_slave class has a unique name identifier ($Instance) displayed in blue text under the three control leds on the graphical representation. The profibus address (BUS ADDRESS) is located to the right of the the RUN-STOP switch.

In the top portion of the graphical representation, there are Power supply (RUN), Bus Fault (BUS-FAULT) and Service Fault (IM-FAULT) indications (LED displays). If the profibus connection is OK, the led Bus Fault is grey otherwise red. If the Power supply is available the RUN led is green otherwise grey.


Figure 1: Graphical Representation
 

Figure 2 shows the Address Popup, which is invoked by selecting the graphical representation. The popup displays the Profibus Address and the actual State of the module. From here, you can also you can reset the Module and open the HTML Documentation.
 

Figure 2: The Address PopUp
 

Figure 3 shows the Error Popup, which is invoked if there any new errors. From this popup, the user can see the actual errors and error messages..
 

Figure 3: Error Popup

Implementation Notes

How to Create the profibus Slave

The first step is to configure a Profibus Slave module. To create a Profibus Slave, take the following steps:

Step 1:
Select the Profibus Slave class and drop an instance of it into the picture window

Step 2:
With the slave object selected, enter a unique instance name and remark in the declaration field of the configuration panel.

When you are naming I/O modules, be sure to assign meaningful names that you will be able to recognize later. For example, the Profibus Slave should at least be given an address number:
 

  • Instance name = Slave_S5
  • Remark= Profibus Slave S5

Figure 4: Declaration for the Profibus Master
 

Step 3:
Open the Server Data tab. Find the resource called "master" in the data list. Click the value field with the left mouse button.

AutomationX automatically detects the master and presents it in a drop down list.

Choose the master name from the list.


Figure 5: Assign the master
 

Step 4:
Find the resource called "address" in the Server Data list. Use the increment/decrement arrows to set the address number of the slave.

For example, if the address of the slave class were 03 you need set the number to 3. When you save the display, you will see this number reflected on the object's "counter".
 

Figure 6: Assign the Profibus Slave address

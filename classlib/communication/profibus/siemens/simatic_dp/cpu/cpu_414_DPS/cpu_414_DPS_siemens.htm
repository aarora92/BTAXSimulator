Diese Dokumentation wurde auf Basis Step7 Version 5.01 erstellt. In anderen Softwareständen
kann es zu leichten Abweichungen in der Darstellung der Dialoge kommen. Die jeweiligen
Abschnitte sind aber auch hier vorhanden.


Installation

Die CP CPU315-2 wird unter Simatic-Step7 auf eine Profilschiene ab Steckplatz 2 eingesetzt.
Dazu wählt man die vorhandenen CPU aus dem Katalog aus (!! Bestellnummer und Version beachten !! ).

 lnstallation | Konfiguration | IS7-Programm


Konfiguration

Durch einen Doppelklick auf den Eintrag "DP-Slave" der Profilschiene wird das
Eigenschaften-Menü geöffnet. Hier wird die CPU benannt und seine Eigenschaften festgelegt.

Allgemein:
Im Dialog Allgemein wird ein beliebiger Name und eventuell noch ein Kommentar vergeben.
Der Abschnitt Allgemein hat für die AX-Kommunikation keinerlei Bedeutung.

Im Abschnitt Schnittstelle ruft man den zugehörigen Eigenschaftsdialog auf.

Eigenschaften Schnittstelle:

Hier wird die Profibusadresse eingestellt.
!! Wichtig !!: Diese Einstellung muss mit der Einstellung unter AutomationX übereinstimmen.

Adressen:
Die Adressen werden von Step7 eingestellt und müssen nicht geändert werden


 

Betriebsart:
Hier muss DP-Slave eingestellt werden. Die Baugruppe ist kein aktiver Teilnehmer
am Profibus Subnetz.


 

Konfiguration:
Im Dialog Konfiguration werden die Eingangs - Ausgangsdaten eingestellt.
In diesem Beispiel, Master Slave Mode, 16 Byte Eingang, 16 Byte Ausgang , Konsistenz über die Gesamte Länge.
Geändert werden die Parameter im Nächsten Dialog ( wird durch doppelklick auf die gewünschte Zeile geöffnet)

Wenn die Input und Output Daten mit dem im automationX projektierten Input und Output Modulen  übereinstimmen, sollte ein zyklischer Betrieb möglich sein.
Um auch Daten zu Senden und zu Empfangen, muss im S7 Prj noch bestimmte read, write, FC`s verwendet werden. (siehe S7-Programm)

S7-Programm

Das S7-Programm muss so erstellt werden, dass die Kommunikations-FC's mindestens
einmal pro Zyklus aufgerufen werden. (z.B. Aufruf der Kommunikations-FC's im OB1)


 

Organisationsbaustein OB82 und OB86:
werden verwendet, damit bei Kommunikationsausfall (CPU - AX) die CPU nicht auf Stop geht

!!Wichtig !!:
Die Datenlänge muss mit der Datenlänge die im PLS projektiert wurde übereinstimmen.

Features

The cpu315_2_di32 class includes 32 digital Inputs and is exclusive for use with
cpu315_DPS class. The module summarize the individual digital inputs to bits in a S7 Datablock who is connected to cpu315_2 with Step7.

Graphical User Interface

The cpu315_2_di32 class represents an S7-datablock from four byte's. The instance name of the module will appear  in large blue text in the middle of the class. The names of the I/O points will appear in small white text just to the right of their corresponding channel. When the module is not connected or not valid, the I/O names will have red background and if Force is on they will have yellow background.

The thirtytwo I/O points are split up into two groups of sixteen represented by a small arc. The arc denote the current state of the channel: grey if inactive, green if active and yellow if they are forced.


Figure 1: Graphical Representation
 
 

The Force PopUp is invoked by selecting the graphical representation. The "Force" button allows each channel to be set to force mode to simulate its value, which is necessary so that PLC-programs can be tested when there is no connection to the I/O. The value of each channel is then simulated/forced with the "ON/OFF" button in the respective section (channel). This button is only active if the "Force" button is set to "Force".

Implementation Notes

Data Transfer will only work correctly when the cpu315_2_di32 Module is connected with at least one Module or Slave.
The position of the Data Modules should be the same as it is defined in the PLC. Each module must be connected in series with the next module. To do this, assign the "Instance" name of the next module into the "Connection" string variable of the current module. If there is no module next in the series, the "connection"field" remains blank.

User Interface

The IM 153-2 graphical representation has a unique name identifier ($Instance) displayed in the middle of the graphical representation. The profibus address is located above the instance name.
In the upper left corner, are indications for Service Fault (SF), Bus Fault (BF) and power supply (ON).
 
 
Indication State Condition
Bus Fault 
(BF)		 red flashing 

red

 Wrong Parameterization - no data transfer between DP Master and IM 153-1 

Wrong Baud rate or DP-Address
Service Fault 
(SF)		 red Configuration is not the same as defined in the PLC
ON green Power is on (will appear grey when no power)
 

Figure 1: Graphical Representation
 

Figure 2 shows the Address Popup, which is invoked by selecting the graphical representation. The popup displays the Profibus Address and the actual State of the module. From here, you can set the Profibus Address, reset the Module and open the HTML Documentation.
 

Figure 2: The Address PopUp
 

Figure 3 shows the Error Popup, which will invoke automatically upon any new error. The popup displays the error that has occurred along with the error message.
 

Figure 3: Error Popup

Implementation Notes

How to Create the profibus Slave

The first step is to configure a Profibus Slave module. To create a Profibus Slave, take the following steps:

Step 1:
Select the Profibus Slave class and drop an instance of it into the picture window

Step 2:
With the slave object selected, enter a unique instance name and remark in the declaration field, in the configuration panel.

When you are naming I/O modules, be sure to assign meaningful names that you will be able to recognize later. For example, the Profibus Slave should at least be given an address number:
 

  • Instance name = IM153_2
  • Remark= Siemens IM153-2

Figure 4: Declaration for the Profibus Master
 
 

Step 3:
Open the Server Data tab. Find the resource called "master" in the data list. Click the value field with the left mouse button.

AutomationX automatically detects the master and present it in a drop down list.

Choose the master name from the list.


Figure 5: Assign the master
 

Step 4:
Find the resource called "address" in the Server Data list. Use the arrow to set the address number of the slave.

For example, if the address of the slave class were 03 you need set the number to 3. When you save thedisplay, you will see this number reflected on the object's "counter".
 

Figure 6: Assign the Profibus Slave address

Graphical User Interface

The et200b_ao4 class represents an actual compact dp slave profibus module. The class name appears on the left top. When the module is not connected or not valid, the classname will flash red. The instance name of the module is shown on the left in the light blue label area. The names of the I/O points will appear in small black text just to the left of their corresponding channel, in the light blue area of the module.

The thirtytwo I/O points are split up into eight groups. Each channel is represented by a small LED. The LEDs denote the current state of the channel: grey if inactive, green if set and yellow if "Force" is on.


Figure 2: Graphical Representation

 In the upper right corner of the graphical representation can be found the power (RUN), bus fault (BF) and diagnosis display (ON).

Figure 2 shows the Address Popup, which is invoked by a mouse click on the slave unit of the graphical representation. The popup displays the profibus address and the actual state of the module. From here, you can change the profibus address, reset the module plus open the HTML documentations.


 Figure 2: The Address PopUp
 

Figure 3 shows the Error Popup, which will invoke if there are any new errors. The popup will display the actual errors plus error messages.


 Figure 3: The Error PopUp
 

The Force PopUp's are invoked by selecting the I/O point area in the light blue label on the graphical representation. The "Norm/Force" button allows each channel to be set to force mode to simulate its value, which is necessary so that PLC-programs can be tested when there is no connection to the I/O. The value of each channel is then simulated/forced with the "ON/OFF" button in the respective section (channel). This button is only active if the "Norm/Force" button is set to "Force".

A red rectangle indicator appears to the right of the I/O name if the module is NOT VALID. The indicator will not be present if the channel is forced or when the module is VALID.

Implementation Notes

How to Create the profibus Slave

The first step is to configure a Profibus Slave module. To create a Profibus Slave, take the following steps:

Step 1:
Select the Profibus Slave class and drop an instance of it into the picture window

Step 2:
With the slave object selected, enter a unique instance name and remark in the declaration field, in the configuration panel.

When you are naming I/O modules, be sure to assign meaningful names that you will be able to recognize later. For example, the Profibus Slave should at least be given an address number:
 

  • Instance name = et200b_ao4

Figure 4: Declaration for the Profibus Master
 

Step3:
Open the Server Data tab. Find the resource called "master" in the data list. Click the value field with the left mouse button.

Automationx automatically detects the master and present it in a drop down list.

Choose the master name from the list.

Figure5: Assign the master
 

Step 4:
Find the resource called "address" in the Server Data list. Use the increment/decrement arrows to set the address number of the slave.

For example, if the address of the slave class were 11 you need set the number to 11. When you save the display, you will see this number reflected on the object`s "counter".

 Figure 6: Assign the Profibus Slave address

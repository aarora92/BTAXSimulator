Features

This digital input/output module "sm323_di8_do8"  includes eight isolated inputs and eight isolated outputs. The sm323_di8_do8 class can be used in a S7-300 or a Profibus DP Slave System (ET200M IM153).

User Interface

The sm323_di8_do8 class represents an actual I/O module. The class name (SM323) appears at the top of the module. When the module is not connected or not valid, this title will flash red.

The instance name of the module will appear in large blue text in the middle of the light blue "label" area. The names of the I/O points will appear in small black text just to the left of their corresponding channel, in the light blue area of the module.

The eight Input and Output points are numbered from 0 - 7, each channel represented by a small rectangle. The rectangles denote the current state of the channel: grey if inactive, green if set and yellow if "Force" is on. If Force is off, the background of the names of the I/O points will turn yellow.

Figure 2: Graphical Representation
 
 

The Force PopUp is invoked by selecting the graphical representation. The "Norm/Force" button allows each channel to be set to force mode to simulate its value, which is necessary so that PLC-programs can be tested when there is no connection to the I/O. The value of each channel is then simulated/forced with the "ON/OFF" button in the respective section (channel). This button is only active if the "Norm/Force" button is set to "Force".

The Reference name of a channel is displayed in blue text (e.g. Q0_0_output). The Remark for that channel is displayed in black text immediately below the Reference name (e.g. Output 0)
 

Figure 3: The Force PopUp
 
 

Figure 4 shows an instance of the Object Explorer in Online mode (click Shift + Right mouse button on the module to invoke the Object Explorer). The Instance name and Remarks are declared in the top portion of the window. A red rectangle indicator appears to the right of the I/O name if the module is NOT VALID. The indicator will not be present if the channel is forced or when the module is VALID.

The variables I0_0 to I0_7  and  Q0_0 to Q0_7 are VAR-GLOBAL. Each live input variable must have a tag name assigned to its "Reference" attribute ( e.g. Q0_4_output) . Also its possible to assign a Remark for every Input and Output Channel ( e.g. Output 4 ).
 

Figure 4: The Object Explorer

Implementation Notes

Data Transfer will only work correctly when the sm323 Module is connected with at least one Module or Slave.  The system will check for an existing module or slave automatically and display the name at the bottom of the graphical representation.

Link the I/O Blocks Together

Once you have placed all of the necessary I/O modules on to the display , you will need to connate them. The I/O objects in AutomationX communicate in series just like real I/O blocks, starting from the Profibus Slave.

Each I/O module has a "Connection" resource in the Server Data list. To connect your I/O modules, start with the Profibus Slave  and type the instance name of the next module (in the sequence ) into the "Connection" field. The "Connection" field  of the last module in the sequence can be blank.

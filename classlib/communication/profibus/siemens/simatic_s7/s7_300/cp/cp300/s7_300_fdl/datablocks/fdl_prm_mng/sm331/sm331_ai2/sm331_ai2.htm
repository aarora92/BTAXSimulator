Features

This analog input module sm331_ai2 includes two inputs with a selectable resolution of 9,12, and 14 bits. The sm331_ai2 class can be used in a S7-300 or a Profibus DP Slave-System (ET200M IM153).

User Interface

The sm331_ai2 class represents an actual I/O module. The class name appears at the top of the module (i.e. SM331). When the module is not connected or not valid, this title will flash red. If Service fault, the rectangle in the upper right corner of the graphical representation lights red otherwise it remains grey.

The instance name of the module will appear  in large blue text in the middle of the light blue "label" area. The names of the I/O points will appear in small black text just to the left of their corresponding channel, in the light blue area of the module.

The two channels are  represented just to the right ot the light blue area. The sliders denote the current state of the channel: grey if inactive, green if set, yellow if "Force" is on and red if the channel deactivated.


Figure 2: Graphical Representation

The Force PopUp is invoked by selecting the graphical representation. The "Norm/Force" button allows each channel to be set to force mode in order to simulate its value. This  is necessary so that PLC programs can be tested while there is no connection to the I/O. The value of each channel is indicated with a slider in the respective channel. The slider is only active if the "Norm/Force" button is set to "Force". If the Module not Valid, the message "Module Not Valid" will flash red in the upper right corner.

The Reference name of the input channel is displayed in blue text (e.g. CH0_input). A descriptive Remark appears immediately below the Reference name (e.g. Input Channel 0).

Figure 3: The Force PopUp

Figure 4 shows an instance of the Object Explorer in Online mode (click Shift + Right mouse button on the module to invoke the Object Explorer). The Instance name and Remark for the module are declared in the top portion of the window. A red rectangle indicator appears to the right of the I/O name (e.g. CH0) if the module is NOT VALID. This indicator will not be present if the channel is forced or if the module is VALID.

The variables CH0 and CH1 are VAR-GLOBAL. Each live input variable must have a tag name assigned to its "Reference" attribute ( e.g. Q0_0_input ) . Also its possible to assign a remark for every Input Channel ( e.g. Input Channel 0 ).

With the Subrange flag enabled, you can set the minscale and maxscale attributes, also you can determine Dimension and Decimal Points for each Channel.

With the "ModuleRange" resource in the Server Data list you get set the module range for each channel (e.g. deact).
 

Figure 4: The Object Explorer

Implementation Notes

Data Transfer will only work correctly when the sm331 Module is connected with at least one Module or Slave.  The system will check for an existing module or slave automatically and display the name at the bottom of the graphical representation.

Link the I/O Blocks Together

Once you have placed all of the necessary I/O modules on to the display, you will need to connect them. The I/O objects in AutomationX communicate in series just like real I/O blocks, starting from the Profibus Slave.

Each I/O module has a "Connection" resource in the Server Data list. To connect your I/O modules, start with the Profibus Slave  and type the instance name of the next module (in the sequence ) into the "Connection" field. The "Connection" field  of the last module in the sequence can be blank.

Features

This analog input/output module sm334_ai4_ao2 includes four inputs and two outputs with a resolution of 8 bit.

It can be used in a S7-300 or in a Profibus DP Slave-System ( ET200M IM153 ).

The module converts the analog process signal to a digital value for the analog inputs and for the analog outputs the other way with the selectet resolution.

Graphical User Interface

The sm334_ai4_ao2 class represents an actual I/O module. The class name appears at the top of the module. When the module is not connected or not valid, the title will flash red. The instance name of the module will appear  in large blue text in the middle of the light blue "label" area. The names of the I/O points will appear in small black text just to the left of their corresponding channel, in the light blue area of the module.

The six I/O points are split up into two groups of eight numbered I0-I3 und O0-O1, each channel represented by a small rectangle. The rectangles denote the current state of the channel: grey if inactive, green if set and yellow if "Force" is on.

Figure 2: Graphical Representation
 
 

The Force PopUp is invoked by selecting the graphical representation. The "Norm/Force" button allows each channel to be set to force mode to simulate its value, which is necessary so that PLC-programs can be tested when there is no connection to the I/O. The value of each channel is then simulated/forced with the "ON/OFF" button in the respective section (channel). This button is only active if the "Norm/Force" button is set to "Force".

Figure 3: The Force PopUp
 
 

Figure 4 shows the Object Explorer in Online mode (click Shift + Right mouse button on the module to invoke the Object Explorer). The user can see where the Instance name and Remarks are declared. A red rectangle indicator appears to the right of the I/O name if the module is NOT VALID. The indicator will not be present if the channel is forced or when the module is VALID.

Figure 4: The Object Explorer

Implementation Notes

Data Transfer will only work correctly when the sm334_ai4_ao2 Module is connected with at least one Module or Slave.  The system will check for an existing module or slave automatically and display the name at the bottom of the graphical representation.

The position of the Data Modules should be the same as it is defined in the PLC. Each module must be connected in series with the next module. To do this, assign the "Instance" name of the next module into the "Connection" string variable of the current module. If there is no module next in the series, the "connection"field" remains blank.

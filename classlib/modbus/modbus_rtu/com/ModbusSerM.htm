Modbus Serial Master

Features

The Modbus Master symbol communicates with Modbus Slaves via the standard serial Interface.
It uses Modbus Client classes as database -> see Configuration.

Graphical User Interface

The ModbusSerM class represents an actual master module. The instance name of the module will appear  in white text on the top right of the symbol. The title will change from green to flashing red if an error occurs.

The symbol is divided into four sections:

- Communication Nr. e.g. COM 1, COM 2, ...

- Client Information
        The Instance name, the Modbus Function, array Nr. and Modbus address of the actual Modbus client is displayed.

- Frame counts:
        Number of sent and received frames.

- State string:
        Informational string.
 

Figure 2: Graphical Representation
 

The PopUp is invoked by selecting the graphical representation. It shows the communication settings and the actual client. To activate the symbol the Select button must be on. On the top of the PopUp the communication settings are displayed.

The Modbus Master uses Modbus Clients (ModbusCL.class) as database.
You must install at least one Modbus Client to communicate with a slave.
Then fill in its Instance name into the Client name array of the Modbus Master.

aX4401_pbc1 Serial Converter


Features

This module represents a RS232 serial converter.

 

The module suports follwing modes:

  • 8-bit 1-stopbits parity no
  • 7-bit 2-stopbits parity no
  • 8-bit 2-stopbits parity no
  • 7-bit 2-stopbits parity even
  • 8-bit 1-stopbits parity even
  • 7-bit 2-stopbits parity even
  • 8-bit 2-stopbits parity even
  • 7-bit 1-stopbits parity odd
  • 8-bit 1-stopbits parity odd
  • 7-bit 2-stopbits parity odd
  • 8-bit 2-stopbits parity odd
  • 7-bit 2-stopbits parity '1'
  • 8-bit 1-stopbits parity '1'
  • 7-bit 2-stopbits parity '1'
  • 8-bit 2-stopbits parity '1'
  • 7-bit 1-stopbits parity '0'
  • 8-bit 1-stopbits parity '0'
  • 7-bit 2-stopbits parity '0'
  • 8-bit 2-stopbits parity '0'

You can use follow baud rates:

  • 300 Baud
  • 600 Baud
  • 1200 Baud
  • 2400 Baud
  • 4800 Baud
  • 9600 Baud
  • 14400 Baud
  • 19200 Baud
  • 28800 Baud
  • 38400 Baud

To config the module push on  the mode name in the graphical representation.

 

Data Send

 

There are two options to send data to the profibus:

  • If the OutputData buffer changes.
  • If the TelegramNrOutSend increases.

You can sellect one of this mode with the bit 3 in the UsrPrmData[5]. Wenn the Telegram was send the TelegramNrInSend value will set to the same value as the TelegramNrOutSend value.

The lenght of the telegram can defined by the TelegramSendLength value, or by seting of one or two end-chars. In this case the converter seaches in the OutputData buffer for this end chars, and send all bytes from the first char inclusive the end chars. If no end-chars can be found, the whole buffer will be send.


Data Reseive

 

If a start-char is defined, the receiving is started when a start-char was receifed. If no start-char is defined the receiving beginns with all chars.

The end of the telegram can identified on two ways:

  • By defining an end-char.
  • By defining a timeout value.

Wenn you use the timeout value, the receiving will stopped if no char was received for this time.

After each receiving of a telegram the TelegramNrInRecive will increased. If you use the telegram numbers you must set the TelegramNrOutRecive valu to the same number.
